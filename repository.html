Repository Pattern

Lớp Repository che giấu nguồn dữ liệu (memory ↔ API) sau cùng interface

1 · Chọn backing store

Đổi store nhưng client không đổi một dòng code — vẫn gọi repo.add() / repo.findById() như cũ.

2 · Thao tác qua repository

🗄 Nội dung store hiện tại

Store rỗng…
Client
UserService / Controller
↓
UserRepository
«interface» — che giấu nguồn dữ liệu
↓
InMemoryUserRepository
Map trong RAM

Dữ liệu đi Client → Repository → DataSource. Client chỉ thấy interface, không biết phía sau là RAM hay API.

💡 Vì sao dùng Repository?

Client chỉ làm việc với interface UserRepository (add / findById / remove). Nó không quan tâm dữ liệu nằm trong RAM, SQL hay gọi REST API.

Nhờ vậy ta thay nguồn dữ liệu (in-memory khi test ↔ API thật khi chạy) chỉ bằng cách tiêm một bản cài khác — không đụng logic nghiệp vụ. Repository tách "lấy dữ liệu ở đâu" khỏi "làm gì với dữ liệu".
