📐 So sánh code

Không Null Object 
if (user != null) {
  d = user.getDiscount();
} else {
  d = 0;          // phải nhớ check!
}
Có Null Object 
d = user.getDiscount();
// luôn an toàn — GuestUser trả 0

Cùng một dòng user.getDiscount() — Null Object trả mặc định an toàn, còn null thì vỡ với TypeError.

💡 Vì sao dùng Null Object?

Thay vì rải if (x != null) khắp nơi, ta tạo một lớp "rỗng nhưng hợp lệ" (GuestUser, NullLogger) cài đủ method và trả về giá trị mặc định vô hại.

Client gọi method như bình thường mà không bao giờ gặp null. Logic kiểm tra null biến mất, code sạch và an toàn hơn.
