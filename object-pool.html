Object Pool

Tái sử dụng object đắt đỏ (connection/đạn) thay vì new liên tục

Thao tác lên pool

acquire() lấy 1 connection ra dùng. Nếu pool còn hàng thì tái dùng; nếu cạn mới phải new (tốn kém). release(c) trả connection về pool để lần sau dùng lại.

📊 Thống kê pool

Đang rảnh (free)0
Đang dùng (in-use)0
Tổng đã new0
Số lần acquire()0

Bấm acquire/release nhiều lần — số object thật sự tạo ra sẽ ít hơn nhiều số lần acquire.

📋 Nhật ký

Pool khởi tạo sẵn 2 connection rảnh…
🗄️ POOL (free)
⚙️ ĐANG DÙNG (in-use)

acquire() kéo connection từ POOL sang vùng dùng; release() đẩy ngược về POOL — cùng một object, không tạo mới.

💡 Vì sao dùng Object Pool?

Tạo một Connection (hay viên đạn, thread, sprite…) tốn kém: mở socket, cấp phát bộ nhớ, bắt tay TLS. Tạo/huỷ liên tục gây giật và rác GC.

Pool giữ sẵn một tập object tái sử dụng. acquire() mượn, release() trả lại sạch sẽ. Object thật sự được new ít hơn rất nhiều so với số lần dùng.
