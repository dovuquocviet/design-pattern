CQRS

Tách Command (ghi) khỏi Query (đọc) — hai mô hình riêng

✍️ Command — phía GHI

Command là ý định thay đổi. Gửi qua bus.send(cmd) → handler ghi vào writeDb rồi phát sự kiện đồng bộ.

🔍 Query — phía ĐỌC

Query chỉ đọc từ readDb đã dựng sẵn — nhanh, không bao giờ chạm vào phía ghi.

🗄️ Hai kho dữ liệu

writeDb · Write Model

Một command đi vào bên trái, được đồng bộ sang phải; query chỉ đọc bên phải — hai con đường tách biệt.

💡 Vì sao dùng CQRS?

Ghi và đọc có nhu cầu rất khác nhau: ghi cần đúng & nhất quán, đọc cần nhanh. CQRS tách thành hai mô hình — Command đổi trạng thái, Query trả dữ liệu — qua hai handler riêng.

Phía ghi phát sự kiện, một projector cập nhật phía đọc (read model đã được "dọn sẵn" tối ưu cho truy vấn). Hai bên scale, tối ưu độc lập.
